## Why use Turborepo

Turborepo helps when one codebase contains multiple apps and shared packages.

- shared builds and caching
- one place for scripts across apps
- faster CI with task reuse
- easier sharing of UI, auth, env, and db packages

## Why not just multiple apps

Separate apps work, but you lose the workspace benefits.

- duplicated setup
- repeated installs and builds
- harder package sharing
- more CI complexity

## How to convert without Turborepo

You can keep the same packages and remove the turbo layer.

1. Replace turbo tasks with plain package scripts.
2. Run scripts per app from the repo root or with a simple shell script.
3. Keep workspace packages if your package manager supports them.
4. Move shared config into normal files instead of turbo pipelines.

## When to drop it

Drop Turborepo if the repo becomes small enough that caching and orchestration no longer matter.
