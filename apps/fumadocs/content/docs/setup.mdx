## Bun

Bun is the default for this repo.

It is a runtime, package manager, and script runner in one binary, so the setup surface is smaller than a Node.js + npm stack. For this codebase, that is the point: fewer moving parts, faster installs, and less time spent waiting on tooling.

Bun can run most Node.js projects with little or no code change, but it is not magic. If a package depends on Node-specific behavior, native addons, or a brittle install script, verify it before assuming parity.

- Bun docs: https://bun.sh/docs
- Package manager docs: https://bun.sh/docs/cli/install
- Skip to [Node fallback](#node-fallback)

## Why Bun Here

- Fast installs and script execution.
- One tool for install, run, and test workflows.
- Bun is stricter about some install-time behavior, which is useful when you want fewer supply-chain surprises.

## Supply Chain Risk

Package install hooks are a real attack surface. If a dependency runs arbitrary code during install, that code runs before you have a chance to inspect the package in normal use.

References:

- npm lifecycle scripts: https://docs.npmjs.com/cli/v11/using-npm/scripts#life-cycle-scripts
- npm dependency security: https://docs.npmjs.com/auditing-package-dependencies-for-security-vulnerabilities
- Axios supply-chain incident writeup: https://socket.dev/blog/axios-supply-chain-attack

## Install Bun

**macOS / Linux:**

```bash
curl -fsSL https://bun.sh/install | bash
```

**Windows PowerShell:**

```powershell
irm bun.sh/install.ps1 | iex
```

**Windows with Scoop:**

```powershell
scoop install bun
```

**Windows with Chocolatey:**

```powershell
choco install bun
```

### Continue with Bun

```bash
bun install
bun run dev
bun run build
bun run check-types
```

## Set Up `.env`

Copy `.env.example` to `.env` at the repo root, then fill in the values for your environment.

The app validates env on startup, which is exactly what you want. If a required value is missing, it should fail immediately instead of breaking later in auth, database, or email code.

### Database

- `DATABASE_URL`: get this from MongoDB Atlas or your local MongoDB server.
- `DATABASE_NAME`: the database name you want the app to use.

### Auth

- `BETTER_AUTH_SECRET`: generate a long random secret, at least 32 characters.
- `CORS_ORIGINS`: comma-separated frontend origins, including local dev and deployed domains.

Get `BETTER_AUTH_SECRET` from a secure secret generator, not from a reusable password.

`CORS_ORIGINS` should include the exact domains that are allowed to talk to the API.

### Email

- `SMTP_HOST`: your mail provider SMTP host.
- `SMTP_PORT`: the SMTP port from that provider.
- `SMTP_USER`: the SMTP username, usually your email address.
- `GMAIL_USER`: the sender email address.
- `GMAIL_PASS`: an app password or SMTP password.

These values usually come from your mail provider settings or Google app-password setup.

### Runtime

- `NODE_ENV`: `development` locally, `production` in deploys.

### Client Values

- `VITE_SERVER_URL`: the API base URL used by the web app.
- Local dev usually points to `http://localhost:3000/api`.
- Production should point to the deployed API domain.

### Why These Matter

- `DATABASE_URL` and `DATABASE_NAME` decide where data is stored.
- `BETTER_AUTH_SECRET` protects auth sessions and token signing.
- `CORS_ORIGINS` prevents random sites from calling your API.
- SMTP values are required for password reset, verification, and other email flows.
- `VITE_SERVER_URL` keeps the frontend pointed at the correct backend.

Do not commit real secrets.

## Node Fallback

Node is the fallback, not the default.

Use it if Bun is unavailable in your environment or you are debugging a Node-only issue.

### Install Node

**macOS with Homebrew:**

```bash
brew install node
```

**macOS / Linux with nvm:**

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
nvm install --lts
nvm use --lts
```

**Windows with winget:**

```powershell
winget install OpenJS.NodeJS.LTS
```

**Windows with Scoop:**

```powershell
scoop install nodejs-lts
```

**Windows with Chocolatey:**

```powershell
choco install nodejs-lts
```

**Ubuntu / Debian:**

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**Fedora:**

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo dnf install -y nodejs
```

### Continue with Node

```bash
npm install
npm run dev
npm run build
npm run check-types
```
